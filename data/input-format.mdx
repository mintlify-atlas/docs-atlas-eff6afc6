All Alpamayo 1.5 inference methods accept a `data` dictionary with a consistent structure. This page describes how to build that dict from raw camera frames and trajectory data.

## Input dict structure

The `data` dict passed to `sample_trajectories_from_data_with_vlm_rollout` or `generate_text` must contain:

```python
model_inputs = {
    "tokenized_data": {          # output of processor.apply_chat_template
        "input_ids":      Tensor,  # (B, seq_len) — token IDs
        "attention_mask": Tensor,  # (B, seq_len) — 1 for real tokens, 0 for padding
        "pixel_values":   Tensor,  # (total_patches, C) — flattened image patches
        "image_grid_thw": Tensor,  # (N_images, 3) — temporal/height/width grid dims
    },
    "ego_history_xyz": Tensor,   # (B, 1, 16, 3)  — past positions in ego frame
    "ego_history_rot": Tensor,   # (B, 1, 16, 3, 3) — past rotations in ego frame
}
```

<Note>
For VQA inference with `generate_text`, only `tokenized_data` is required. `ego_history_xyz` and `ego_history_rot` are not needed.
</Note>

## Building the input

```python
import torch
from alpamayo1_5 import helper
from alpamayo1_5.load_physical_aiavdataset import load_physical_aiavdataset
from alpamayo1_5.models.alpamayo1_5 import Alpamayo1_5

# 1. Load dataset sample
data = load_physical_aiavdataset(clip_id="030c760c-ae38-49aa-9ad8-f5650a545d26", t0_us=5_100_000)

# 2. Load model and get processor
model = Alpamayo1_5.from_pretrained("nvidia/Alpamayo-1.5-10B", dtype=torch.bfloat16).to("cuda")
processor = helper.get_processor(model.tokenizer)

# 3. Build the chat message (images + trajectory history + optional nav)
messages = helper.create_message(
    frames=data["image_frames"].flatten(0, 1),  # (N_cameras*4, 3, H, W)
    camera_indices=data["camera_indices"],
    nav_text=None,  # or "Turn left in 40m" for navigation guidance
)

# 4. Tokenize
inputs = processor.apply_chat_template(
    messages,
    tokenize=True,
    add_generation_prompt=False,
    continue_final_message=True,
    return_dict=True,
    return_tensors="pt",
)

# 5. Assemble final input dict
model_inputs = {
    "tokenized_data": inputs,
    "ego_history_xyz": data["ego_history_xyz"],  # (1, 1, 16, 3)
    "ego_history_rot": data["ego_history_rot"],  # (1, 1, 16, 3, 3)
}

# 6. Move to device
model_inputs = helper.to_device(model_inputs, "cuda")
```

## Trajectory tensor format

All ego trajectory tensors use **shape `(B, n_traj_group, T, ...)`**:

| Tensor | Shape | Description |
|---|---|---|
| `ego_history_xyz` | `(B, 1, 16, 3)` | Past 1.6s XYZ positions in ego frame at t0 |
| `ego_history_rot` | `(B, 1, 16, 3, 3)` | Past 1.6s rotation matrices in ego frame at t0 |

Coordinates are in the ego vehicle's local frame at t0, where the origin is the vehicle position at t0.

## Device and dtype

Use `helper.to_device(data, device, dtype)` to recursively move all tensors in the dict to the target device and dtype. During inference, use `torch.autocast` to run in bfloat16:

```python
model_inputs = helper.to_device(model_inputs, "cuda")

with torch.autocast("cuda", dtype=torch.bfloat16):
    pred_xyz, pred_rot = model.sample_trajectories_from_data_with_vlm_rollout(
        data=model_inputs,
        num_traj_samples=6,
    )
```

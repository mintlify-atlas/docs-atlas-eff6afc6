---
title: "Trajectory prediction with VLM reasoning"
description: "Run Alpamayo 1.5 trajectory prediction to generate 64 future waypoints at 10 Hz alongside interpretable chain-of-causation reasoning text."
---

Alpamayo 1.5 trajectory prediction is a dual-output process: a single call to `sample_trajectories_from_data_with_vlm_rollout` simultaneously produces chain-of-causation (CoC) reasoning text describing what the model observes in the driving scene, and trajectory waypoints (`pred_xyz`, `pred_rot`) representing predicted future motion. Both outputs share the same forward pass — the VLM reasoning directly conditions the diffusion-based trajectory sampler.

## Parameters

<ParamField body="data" type="dict" required>
  Input data dict containing `tokenized_data` (processor output), `ego_history_xyz`, and `ego_history_rot`. Build this dict using `helper.create_message` and `processor.apply_chat_template`.
</ParamField>

<ParamField body="top_p" type="float" default="0.98">
  Nucleus sampling parameter for VLM text generation. Tokens are sampled from the smallest set whose cumulative probability exceeds `top_p`.
</ParamField>

<ParamField body="top_k" type="int">
  Top-k sampling parameter. When set, restricts sampling to the `top_k` most probable tokens. Mutually exclusive with `top_p` in practice; leave unset to use nucleus sampling only.
</ParamField>

<ParamField body="temperature" type="float" default="0.6">
  Sampling temperature for the VLM. Lower values produce more deterministic text; higher values increase diversity.
</ParamField>

<ParamField body="num_traj_samples" type="int" default="6">
  Number of trajectory samples to draw per input. Higher values yield more diverse trajectory hypotheses and improve minADE at the cost of additional VRAM.
</ParamField>

<ParamField body="num_traj_sets" type="int" default="1">
  Number of independent trajectory sets to generate. Output tensors have a leading `num_traj_sets` dimension.
</ParamField>

<ParamField body="return_extra" type="bool" default="false">
  When `True`, the method returns a third element — a dict containing `"cot"` and `"meta_action"` string arrays of shape `(B, num_traj_sets, num_traj_samples)`.
</ParamField>

## Return values

- **`pred_xyz`** — `torch.Tensor` of shape `(B, num_traj_sets, num_traj_samples, num_future_steps, 3)`. Predicted XYZ waypoints in the ego vehicle frame at 10 Hz for 6.4 seconds (64 steps).
- **`pred_rot`** — `torch.Tensor` of shape `(B, num_traj_sets, num_traj_samples, num_future_steps, 3, 3)`. Predicted rotation matrices for each waypoint.
- **`extra`** _(only when `return_extra=True`)_ — `dict` mapping `"cot"` and `"meta_action"` to `np.ndarray` of shape `(B, num_traj_sets, num_traj_samples)`.

## Full example

```python trajectory_prediction.py
import numpy as np
import torch
from alpamayo1_5 import helper
from alpamayo1_5.load_physical_aiavdataset import load_physical_aiavdataset
from alpamayo1_5.models.alpamayo1_5 import Alpamayo1_5

clip_id = "030c760c-ae38-49aa-9ad8-f5650a545d26"
data = load_physical_aiavdataset(clip_id, t0_us=5_100_000)

model = Alpamayo1_5.from_pretrained("nvidia/Alpamayo-1.5-10B", dtype=torch.bfloat16).to("cuda")
processor = helper.get_processor(model.tokenizer)

messages = helper.create_message(
    frames=data["image_frames"].flatten(0, 1),
    camera_indices=data["camera_indices"],
)
inputs = processor.apply_chat_template(
    messages,
    tokenize=True,
    add_generation_prompt=False,
    continue_final_message=True,
    return_dict=True,
    return_tensors="pt",
)
model_inputs = {
    "tokenized_data": inputs,
    "ego_history_xyz": data["ego_history_xyz"],
    "ego_history_rot": data["ego_history_rot"],
}
model_inputs = helper.to_device(model_inputs, "cuda")

torch.cuda.manual_seed_all(42)
with torch.autocast("cuda", dtype=torch.bfloat16):
    pred_xyz, pred_rot, extra = model.sample_trajectories_from_data_with_vlm_rollout(
        data=model_inputs,
        top_p=0.98,
        temperature=0.6,
        num_traj_samples=16,
        return_extra=True,
    )

print("Chain-of-Causation:\n", extra["cot"][0])
# pred_xyz shape: (1, 1, 16, 64, 3)
# pred_rot shape: (1, 1, 16, 64, 3, 3)
```

## Evaluating with minADE

minADE (minimum Average Displacement Error) measures the L2 distance in XY between the ground-truth future trajectory and the closest predicted trajectory across all samples.

```python evaluate_minade.py
gt_xy = data["ego_future_xyz"].cpu()[0, 0, :, :2].T.numpy()
pred_xy = pred_xyz.cpu().numpy()[0, 0, :, :, :2].transpose(0, 2, 1)
diff = np.linalg.norm(pred_xy - gt_xy[None, ...], axis=1).mean(-1)
min_ade = diff.min()
print("minADE:", min_ade, "meters")
```

<Note>
  Increasing `num_traj_samples` provides more diverse trajectories and typically improves minADE, but requires more VRAM. See [Hardware Requirements](/hardware-requirements) for memory requirements at different sample counts.
</Note>
